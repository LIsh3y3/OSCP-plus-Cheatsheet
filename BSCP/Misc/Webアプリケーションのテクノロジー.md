# ASP

## 基本情報

- ASP：(Active Server Pagesの略)
	- 由来はWebサーバー上で動的にWebページを生成するため
- Microsoftが開発したWebアプリケーション開発フレームワーク

---

## システム構成情報の推測

- ASPを使っている＝Windowsの可能性が高い(Linuxも可能だが)
	- なぜなら、ASP.NET + IIS + SQL Server という「Microsoftスタック」が主流だから
- 💡`.aspx` + `VIEWSTATE` + `SqlClient` + `System.Data.SqlClient.SqlException` のような情報が重なると、ほぼ*Windows + MSSQL + ASP.NET + IIS* のセットと見なせる

---

## ASP特有のパラメタ

### `__VIEWSTATE`

|           | 説明                                                                             |
| --------- | ------------------------------------------------------------------------------ |
| 意味        | ASP.NETがクライアント側にサーバー状態（ページの状態）を保存するためのBase64エンコード済み文字列                         |
| 役割        | ページ間のデータ保持<br>例えば、ポストバック時に入力値やUIの状態を復元できるようにする                                 |
| セキュリティの観点 | ・デフォルトでは暗号化されていないが、改ざん検出用のMACが含まれる（`ViewStateUserKey`設定でCSRF対策可能）。             |
| 攻撃        | ViewStateコードインジェクション攻撃<br>　└悪意ある.NETオブジェクトを埋め込み、サーバー側でデシリアライズさせることによりコードを実行させる |

---

### `__VIEWSTATEGENERATOR`

|     | 説明                                                                   |
| --- | -------------------------------------------------------------------- |
| 意味  | `__VIEWSTATE`を生成するロジックの検証用トークン。                                      |
| 役割  | ViewStateを生成したページと一致するかを確認することで、クロスページViewStateの悪用を防ぐ<br>	→CSRF対策となる |

---

### `__EVENTVALIDATION`

|           | 説明                                                      |
| --------- | ------------------------------------------------------- |
| 意味        | サーバー側で有効とされるイベントの一覧をハッシュ化したもの。                          |
| 役割        | ユーザーが意図しないイベント（例：HTMLを書き換えて存在しないボタンを追加）をサーバーで拒否するための検証。 |
| セキュリティの観点 | 通常改ざんできないが、古いバージョンや設定ミスで攻撃可能なことがある。                     |

---

### `ctl00$ContentPlaceHolder1$UsernameTextBox`

|     | 説明                                                                                                                                                                                         |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 意味  | ASP.NET WebFormsのサーバーコントロール名                                                                                                                                                               |
| 構造  | ctl00: ページ全体のマスター(最上位)コントロール<br> └ ContentPlaceHolder1: マスター内のプレースホルダー<br>     └ UsernameTextBox: 実際のユーザー名入力フィールド<br><br>※マスターとは、ページのテンプレートのことで、`ContentPlaceHolder`はテンプレーの中身で可変な場所のこと<br> |
| 示唆  | このパラメタは、ASP.NET WebFormsであることを示す<br>（MVCやASP.NET Coreではこのようなパラメータ名は出てこない）<br>　（ASP.NET CoreはクラウドベースWebアプリ開発時に使用）<br>                                                                       |
